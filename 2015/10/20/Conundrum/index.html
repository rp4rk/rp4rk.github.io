Conundrum

Conundrum is an addictive time waster built on Meteor and AngularJS.

Project Information

Conundrum is an addictive time waster, players can submit their conundrums and vote on the submissions of other players. Each conundrum consists of a proposal and an accompanying setback. The goal of each conundrum should be to create an interesting situation where players have to think about their choices, and perhaps agree to less than ideal conditions to get what they want.

Conundrum is developed using Meteor and AngularJS, with MongoDB serving as the NoSQL database. The reason for using the Meteor stack is due to the reactivity it provides, allowing users to see in real time the effect of their votes.

Project Goals

Conundrum Aims to be:

  • A casual timewaster letting players jump in and start straight away.

  • Community building, with features planned to emphasize contributing and encourage content moderation.

  • Fully responsive and cross platform, working on everything from vertical monitors to phones.

  • Device adaptive, with plans for swipe functionality on touch devices.

  • Most importantly, fun.

Features & Performance

Animation performance is extremely important, especially in games where it can be used to enhance the feeling of agency. Conundrum utilizes Meteor to provide a reactive feedback system which lets the user see in real time what effect their vote has on the standing of each Conundrum.

Animations are powered using CSS3, with careful consideration to not trigger content reflows. 60 FPS is the benchmark and it has been hit on all animations in the project so far.
